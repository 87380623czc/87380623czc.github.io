git学习笔记

Created2024-03-26|Updated2024-03-26
|Post Views:

文件三个分区：

  1. 工作区（Working Directory）
  2. 暂存区（Staging Area/Index）
  3. 本地仓库（Local Directory）也就是 repo

文件四个状态：

  1. 未跟踪（Untracked）
  2. 未修改（Unmodified）
  3. 已修改 (Modified)
  4. 已暂存 (Staged)

几个关键的命令：

  1. git init 创建仓库
  2. git status 查看仓库状态
  3. git add 添加到暂存区
  4. git commit 提交
  5. git commit -m 编辑提交信息
  6. git log 查看仓库提交历史记录。后续的–online 可以查看简洁版本的记录
  7. git reset
  8. –soft 保留工作区和暂存区内容
  9. –hard 不保留工作区和暂存区内容
  10. –mixed 保留工作区，不保留暂存区
  11. git diff 比较不同版本的差异，默认为工作区与暂存区，git diff HEAD 比较工作区和版本库的差异
  12. 删除文件的命令：
  13. 删除文件，git add 将工作区的删除操作同步到暂存区
  14. git rm 直接在工作区和暂存区删除。需要 git commit 才能在版本库中也删除
  15. git ignore 忽略同步的文件，包括。需要创建. gitignore 文件。git 的 github 网站上提供了各种语言的. gitignore 模板，可以在创建时使用。
  16. 系统或软件自动产生的文件
  17. 编译产生的中间文件和结果文件
  18. 运行产生的日志、缓存和临时文件
  19. 设计身份，秘密，token 文件

远程仓库


github 中一个项目就是一个仓库
远程仓库有两种方式：HTTPS 和 SSH

  1. HTTPS 将本地仓库 push 到远程时需要验证用户名和密码
  2. SSH 不需要验证，但是需要公钥，更安全方便。有 x 和 x.pub 两个文件，x 为私钥文件，x.pub 为公钥文件

Branch 分支

目的是为了不同的人可以同时修改代码，但保持 main 分支的可靠性
创建方法为 git branch name
切换分支方法为 git switch name 来切换所在分支
